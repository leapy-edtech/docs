## Operações Comuns

### Registrar Presença Manualmente

Quando o talento não conseguiu fazer check-in pelo sistema:

1. Acesse o evento no Backoffice
2. Vá para **Attendances**
3. Clique em **+ Criar**
4. Selecione o talento
5. Preencha horários de check-in/checkout
6. Defina método como `manual`
7. Adicione observação explicativa

### Corrigir Presença Incorreta

1. Localize a presença em **Attendances**
2. Edite os campos necessários
3. Atualize o `status` se necessário
4. Registre o motivo da correção em `notes`

### Encerrar Evento Manualmente

Se o evento não foi encerrado automaticamente:

1. Acesse o evento
2. Altere `status` para `closed`
3. Verifique se todos os check-outs foram processados

```bash
# Processar check-outs pendentes
inngest.send({
  name: "presences/process-pending-checkouts",
  data: { event_id: "..." }
})
```

### Reenviar Convites

Para reenviar convites não recebidos:

```bash
inngest.send({
  name: "presences/resend-invites",
  data: { 
    event_id: "...",
    talent_ids: ["..."] // opcional, se vazio reenvia para todos
  }
})
```

## Troubleshooting

### Check-in não funciona

**Possíveis causas:**

1. Evento ainda não iniciou (fora da janela)
2. QR Code expirado ou inválido
3. Localização fora do raio permitido
4. Talento não está na lista de participantes

**Diagnóstico:**

```sql
-- Verificar se talento é participante
SELECT * FROM event_participants 
WHERE event_id = '...' AND talent_id = '...';

-- Verificar status do evento
SELECT status, start_datetime, end_datetime 
FROM events WHERE id = '...';
```

### Presença não foi contabilizada

**Possíveis causas:**

1. Check-out não foi realizado
2. Tempo de permanência insuficiente
3. Job de processamento falhou

**Solução:**

1. Verificar se existe registro em `attendances`
2. Verificar `checkout_at` e `duration_minutes`
3. Recalcular presença manualmente se necessário

### QR Code não escaneia

**Possíveis causas:**

1. Código expirado
2. Iluminação inadequada
3. Câmera do dispositivo com problema

**Soluções:**

1. Gerar novo QR Code para o evento
2. Usar método alternativo (link ou manual)

## Detectar Pipeline Zoom Parado

O pipeline cron Make + flows Directus que materializa `report_presenca_zoom` está descrito em
[Pipeline Zoom → Backoffice](/documentation/domains/presences/pipeline-zoom-backoffice). A
forma mais simples de detectar parada é olhar `report_presenca_logs`:

```sql
-- Saúde do pipeline nos últimos 14 dias
SELECT
  date_trunc('day', meeting_date)::date AS dia,
  count(*) AS runs,
  count(*) FILTER (WHERE success = true) AS ok,
  count(*) FILTER (WHERE success = false) AS fail
FROM report_presenca_logs
WHERE meeting_date >= now() - interval '14 days'
GROUP BY 1
ORDER BY 1 DESC;
```

**Sinais de pipeline parado:**

| Sinal | Interpretação |
|---|---|
| Zero runs por > 24h em dia útil | Cenário Make `[2546826] Presenças Zoom` provavelmente não disparou. Checar UI Make. |
| Volume < 5 runs em dia útil | Algumas turmas não foram coletadas. Investigar `Turmas.zoom_id` / `link_do_zoom`. |
| Picos de `success = false` | Zoom API com throttling ou credencial expirada. Tech precisa renovar. |

**Caso real (25-28/05/2026):** quatro dias úteis sem runs novos em `report_presenca_logs`.
Detecção via essa query, recuperação posterior via re-execução manual do cenário
`[2547176] Presenças Retroativas` + flow `[PARTE 0] trigger presencas flow`.

### Procedimento de Escalation

1. **PL detecta** sintoma (jovens reportam ausência indevida; relatório bate "vazio").
2. Roda a query acima → se `runs = 0` em dia útil, confirma parada do pipeline.
3. **Escala pra time tech** com: janela afetada (datas), turmas impactadas, último run com
   sucesso.
4. **Time tech** verifica Make UI, status do cenário, credenciais Zoom; executa
   `Presenças Retroativas` pra janela; aciona flow manual `[PARTE 0]`.
5. **PL confirma** pós-fato consultando `report_presenca_zoom` por jovem afetado.

PL não executa re-execução manual diretamente; sempre via tech.

<Note>
  Considerar automação de alerta "0 runs em dia útil" via Sentry, cron de self-check
  ou Inngest scheduled function — reduz tempo de detecção de horas pra minutos.
</Note>
