## Visão Geral

As métricas do Matchmaker são extraídas da tabela `matchmaker_search_logs` e dos logs do Agent System. Permitem avaliar performance técnica, eficácia do matching e adoção pelos usuários.

---

## Métricas de Performance

### Tempo de Processamento

| Métrica | Query | Alvo |
|---------|-------|------|
| **Tempo médio** | `AVG(execution_time_ms)` | < 5.000ms |
| **P95** | `PERCENTILE_CONT(0.95) WITHIN GROUP (ORDER BY execution_time_ms)` | < 15.000ms |
| **P99** | `PERCENTILE_CONT(0.99) WITHIN GROUP (ORDER BY execution_time_ms)` | < 30.000ms |
| **Timeout rate** | Buscas com `error_message LIKE '%timeout%'` / total | < 2% |

```sql
SELECT
  AVG(execution_time_ms) as media,
  PERCENTILE_CONT(0.5) WITHIN GROUP (ORDER BY execution_time_ms) as p50,
  PERCENTILE_CONT(0.95) WITHIN GROUP (ORDER BY execution_time_ms) as p95,
  PERCENTILE_CONT(0.99) WITHIN GROUP (ORDER BY execution_time_ms) as p99,
  MAX(execution_time_ms) as maximo
FROM matchmaker_search_logs
WHERE success = true
  AND date_created > NOW() - INTERVAL '7 days';
```

### Tempo por Camada de Role

Estimativa baseada nos tempos esperados:

| Camada | Tempo esperado | Impacto no total |
|--------|---------------|------------------|
| Exact match | ~1-2ms | Mínimo |
| Alias match | ~1-2ms | Mínimo |
| Trigram match | ~10-30ms | Baixo |
| LLM inference | ~500-2000ms | Alto |

<Tip>
  Se o tempo médio está alto, investigue quantas buscas estão caindo na camada 4 (LLM). Adicionar aliases para roles comuns pode reduzir significativamente.
</Tip>

## Métricas de Eficácia

### Taxa de Sucesso

```sql
SELECT
  COUNT(CASE WHEN success = true THEN 1 END)::float / COUNT(*) * 100 as taxa_sucesso,
  COUNT(CASE WHEN success = false THEN 1 END) as total_falhas,
  COUNT(*) as total_buscas
FROM matchmaker_search_logs
WHERE date_created > NOW() - INTERVAL '30 days';
```

| Métrica | Cálculo | Alvo |
|---------|---------|------|
| **Taxa de sucesso** | Buscas bem-sucedidas / total | > 90% |
| **Taxa de resultado vazio** | Buscas com `candidates_found = 0` / total | < 20% |

### Candidatos por Busca

```sql
SELECT
  AVG(candidates_found) as media_candidatos,
  PERCENTILE_CONT(0.5) WITHIN GROUP (ORDER BY candidates_found) as mediana,
  MIN(candidates_found) as minimo,
  MAX(candidates_found) as maximo
FROM matchmaker_search_logs
WHERE success = true
  AND date_created > NOW() - INTERVAL '30 days';
```

| Métrica | Cálculo | Alvo |
|---------|---------|------|
| **Média de candidatos** | `AVG(candidates_found)` | 5-15 |
| **Busca sem resultado** | `candidates_found = 0` | < 15% |

### Distribuição de Estratégias

Para analisar quais estratégias estão sendo usadas (requer parse do JSONB):

```sql
SELECT
  response_data->'flow_decision'->>'strategy' as estrategia,
  COUNT(*) as total,
  AVG(candidates_found) as media_candidatos,
  AVG(execution_time_ms) as tempo_medio
FROM matchmaker_search_logs
WHERE success = true
  AND response_data IS NOT NULL
  AND date_created > NOW() - INTERVAL '30 days'
GROUP BY estrategia
ORDER BY total DESC;
```

| Estratégia | Esperado | Indica |
|------------|----------|--------|
| `exact_match` | ~40-60% | Base taxonômica boa |
| `llm_match` | ~30-50% | Queries diversas |
| `impossible` | < 5% | Queries muito vagas |

## Métricas de Normalização

Analisar como as skills estão sendo normalizadas:

```sql
SELECT
  (response_data->'normalization_result'->'stats'->>'exact')::int as exatas,
  (response_data->'normalization_result'->'stats'->>'alias')::int as aliases,
  (response_data->'normalization_result'->'stats'->>'synonym')::int as sinonimos,
  (response_data->'normalization_result'->'stats'->>'embedding')::int as embedding,
  (response_data->'normalization_result'->'stats'->>'new')::int as novas
FROM matchmaker_search_logs
WHERE success = true
  AND response_data->'normalization_result' IS NOT NULL
  AND date_created > NOW() - INTERVAL '7 days';
```

| Camada | Alvo | Indica problema se |
|--------|------|-------------------|
| **Exact** | > 50% | < 30% (base desatualizada) |
| **Alias** | 10-20% | > 40% (muitos aliases, consolidar base) |
| **Embedding** | 5-15% | > 30% (base com gaps) |
| **New** | < 10% | > 25% (skills muito específicas ou base defasada) |

## Métricas de Adoção

### Uso por Período

```sql
SELECT
  DATE_TRUNC('day', date_created) as dia,
  COUNT(DISTINCT user_id) as usuarios_unicos,
  COUNT(*) as total_buscas
FROM matchmaker_search_logs
WHERE date_created > NOW() - INTERVAL '30 days'
GROUP BY dia
ORDER BY dia DESC;
```

### Top Usuários

```sql
SELECT
  du.first_name || ' ' || du.last_name as usuario,
  COUNT(*) as total_buscas,
  COUNT(DISTINCT DATE_TRUNC('day', msl.date_created)) as dias_ativos
FROM matchmaker_search_logs msl
JOIN directus_users du ON du.id = msl.user_id
WHERE msl.date_created > NOW() - INTERVAL '30 days'
GROUP BY du.id, du.first_name, du.last_name
ORDER BY total_buscas DESC
LIMIT 10;
```

### Rate Limit Hits

```sql
SELECT
  DATE_TRUNC('day', date_created) as dia,
  COUNT(*) as rate_limits
FROM matchmaker_search_logs
WHERE error_message LIKE '%rate%' OR error_message LIKE '%limite%'
  AND date_created > NOW() - INTERVAL '30 days'
GROUP BY dia
ORDER BY dia DESC;
```
