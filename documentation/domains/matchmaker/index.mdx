## Visão Geral

O Matchmaker é uma feature **chat-based** que permite a usuários RH encontrar talentos compatíveis com uma vaga ou perfil desejado. O usuário pode descrever o que precisa em linguagem natural — seja colando uma URL de vaga, descrevendo um cargo ou listando skills — e o sistema executa um pipeline de IA em 5 passos para retornar candidatos rankeados por percentual de compatibilidade.

## Pipeline de Processamento

O Orchestrator executa **5 passos sequenciais**, cada um com responsabilidade bem definida:

```mermaid
flowchart LR
    P1[1. InputProcessor] --> P2[2. FlowEngine]
    P2 --> P3[3. SkillProcessor]
    P3 --> P4[4. SkillNormalizer]
    P4 --> P5[5. CandidateMatcher]
```

| Passo | Componente | Responsabilidade |
|-------|-----------|------------------|
| 1 | **InputProcessor** | Classifica input (texto/URL/misto), faz scraping se URL, extrai role/skills/seniority via LLM |
| 2 | **FlowDecisionEngine** | Decide estratégia de matching (EXACT_MATCH, LLM_MATCH, IMPOSSIBLE) e flags de controle |
| 3 | **SkillProcessor** | Processa e infere skills adicionais via LLM quando necessário |
| 4 | **SkillNormalizer** | Normaliza skills para IDs do banco em 4 camadas (exact, alias, embedding, new) |
| 5 | **CandidateMatcher** | Busca candidatos por skills, calcula match_score e opcionalmente re-rankeia via LLM |

## Tipos de Input Suportados

| Tipo | Exemplo | Processamento |
|------|---------|---------------|
| **Texto livre** | "Desenvolvedor Python sênior com AWS" | LLM extrai role, skills e seniority |
| **URL de vaga** | `https://linkedin.com/jobs/view/123` | Scraping + LLM extrai do conteúdo |
| **Misto** | "Quero alguém para essa vaga https://..." | Detecta URL, faz scraping, combina com texto |
| **Apenas skills** | "Python, React, Docker, AWS" | Usa skills diretamente sem busca de role |

## Estratégias de Matching

```mermaid
stateDiagram-v2
    [*] --> TemRole
    TemRole --> BuscaBase: Sim
    TemRole --> TemArea: Não

    BuscaBase --> EXACT_MATCH: Role encontrado
    BuscaBase --> LLM_MATCH: Role não encontrado

    TemArea --> LLM_MATCH: Sim
    TemArea --> TemSkills: Não

    TemSkills --> LLM_MATCH: Sim
    TemSkills --> IMPOSSIBLE: Não
```

| Estratégia | Quando | Comportamento |
|------------|--------|---------------|
| **EXACT_MATCH** | Role encontrado na base taxonômica | Ranking por quantidade de skills em comum |
| **LLM_MATCH** | Role não encontrado, ou apenas área/skills | LLM infere skills e/ou re-rankeia candidatos |
| **IMPOSSIBLE** | Nenhuma informação útil extraída | Retorna erro com orientação ao usuário |

## Tecnologias

| Componente | Tecnologia | Uso |
|------------|------------|-----|
| **Backend IA** | FastAPI (Python) | API do Agent System, Orchestrator |
| **Proxy/Auth** | Directus (Node.js) | Autenticação, rate limiting, logging |
| **LLM** | OpenAI GPT-4o-mini | Extração de role/skills, inferência, re-ranking |
| **Database** | PostgreSQL (Supabase) | Base taxonômica (63k roles, 32k skills) |
| **Busca fuzzy** | pg_trgm | Busca por similaridade em roles e skills |
| **Frontend** | Next.js 14 | Interface de chat com streaming SSE |
| **Streaming** | Server-Sent Events | Feedback em tempo real durante processamento |
| **Scraping** | httpx + BeautifulSoup | Extração de conteúdo de URLs de vagas |
