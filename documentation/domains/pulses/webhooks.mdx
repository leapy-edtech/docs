## Origem dos Eventos: AutomaĂ§ĂŁo com Hooks CRON

Os eventos Inngest sĂŁo disparados de duas formas principais:

1. **Hook CRON â†’ Endpoint Workflow â†’ Evento** (fluxo recomendado)
2. **Hook CRON â†’ Evento Direto** (para eventos simples)

### Fluxo de AutomaĂ§ĂŁo


**Fluxo 2: Hook disparando Evento Direto (jovens)**

```javascript
// extensions/hooks/src/hook-cron-change-status-pulso-jovens/index.js
// Dois blocos schedule no mesmo hook: agendadoâ†’pendente e pendenteâ†’vencido
// Nota: nĂŁo usa pulsosSettingsService â€” filtra diretamente por campos da collection pulsos_jovens
export default ({ schedule }, { services, getSchema }) => {
  const scheduleStatusUpdate = ({ dateField, statusFilter, eventName, message, extraConditions = [] }) => {
    schedule("0 5 * * *", async () => {
      const today = new Date().toISOString();
      const pulsosService = new ItemsService("pulsos_jovens", { schema: await getSchema() });
      const pulsos_jovens = await pulsosService.readByQuery({
        fields: ["id", "status", dateField, "jovem_id.id"],
        filter: { _and: [{ [dateField]: { _lte: today } }, { status: statusFilter }, ...extraConditions] },
      });
      if (pulsos_jovens.length > 0)
        await sendEvent({ name: eventName, data: { message, ids: pulsos_jovens.map(p => p.id) } });
    });
  };

  // Bloco 1: agendado â†’ pendente (data_aplicacao <= hoje)
  scheduleStatusUpdate({ dateField: "data_aplicacao", statusFilter: { _eq: "agendado" }, eventName: "backoffice/pulsos-jovens.update-to-pendente", message: "Atualizar pulsos agendados para pendente" });

  // Bloco 2: pendente â†’ vencido (data_vencimento <= hoje, sem data_resposta)
  scheduleStatusUpdate({ dateField: "data_vencimento", statusFilter: { _nin: ["respondido", "vencido"] }, eventName: "backoffice/pulsos-jovens.update-to-vencidos", message: "Atualizar pulsos pendentes para vencido", extraConditions: [{ data_resposta: { _null: true } }] });
};
```

**Fluxo 2: Hook disparando Evento Direto (lideranĂ§as)**

```javascript
// extensions/hooks/src/hook-cron-change-status-pulso-lideranca/index.js
// Usa pulsosSettingsService para filtrar por perfil habilitado por account (aprendiz/estagiĂˇrio/lideranĂ§a)

  schedule("0 5 * * *", async () => {

    const pulsosToProcess = await pulsosSettingsService.filterPulsosLiderancasByAccess(normalized, "lideranca_id");
    await sendEvent({ name: "backoffice/pulsos.change_status", data: { pulsos: pulsosToProcess } });
  });

  // Bloco 2: pendente â†’ vencido (data_vencimento <= hoje, sem data_resposta)
  schedule("0 5 * * *", async () => {
    const pulsos = await pulsosService.readByQuery({ filter: { status: { _nin: ["respondido", "vencido"] }, data_vencimento: { _lte: today }, data_resposta: { _null: true } } });
    const normalized = pulsos.map(p => ({ ...p, lideranca_id: p.jovem_id?.lideranca_id }));
    const pulsosToProcess = await pulsosSettingsService.filterPulsosLiderancasByAccess(normalized, "lideranca_id");
    await sendEvent({ name: "backoffice/pulsos.vencidos_lideranca", data: { pulsos: pulsosToProcess } });
  });
};
```

**Endpoint Workflow disparando Evento**

```javascript
// extensions/endpoints-workflows/src/pulsos/index.js
router.get("/send-liberacao-pulsos-rh-to-rh", async (req, res) => {
  const { status = "agendado", send_event = true } = req.query;

  // 1. Consultar dados
  const pulsos = await services.ItemsService("pulsos_rh").readByQuery({
    filter: { status: { _eq: status } },
  });

  // 2. Disparar evento Inngest
  if (send_event && pulsos.length > 0) {
    await fetch(env.INNGEST_EVENT_URL, {
      method: "POST",
      headers: {
        Authorization: `Bearer ${env.INNGEST_EVENT_KEY}`,
        "Content-Type": "application/json",
      },
      body: JSON.stringify({
        name: "backoffice/pulsos_rh/liberacao-to-rh.send",
        data: {
          pulso_ids: pulsos.map((p) => p.id),
          count: pulsos.length,
        },
      }),
    });
  }

  res.json({ data: { pulsos_encontrados: pulsos.length } });
});
```

### Quando Usar Cada Fluxo?

| CritĂ©rio         | Hook â†’ Endpoint â†’ Evento                                                           | Hook â†’ Evento Direto                          |
| ---------------- | ---------------------------------------------------------------------------------- | --------------------------------------------- |
| **Quando usar**  | Precisa consultar dados, validar regras, processar lĂłgica antes de disparar evento | Evento simples, sem necessidade de consultas  |
| **Vantagens**    | Controle fino, logging, validaĂ§ĂŁo, testĂˇvel via HTTP                               | Simples, direto, menos overhead               |
| **Desvantagens** | Mais complexo, mais cĂłdigo                                                         | Limitado, sem lĂłgica customizada              |
| **Exemplo**      | Enviar notificaĂ§Ăµes apenas para pulsos com status "agendado"                       | Atualizar status de todos os pulsos pendentes |
