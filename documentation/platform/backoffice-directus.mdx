|------------|-----------|

## Usando a API

### REST API

```bash
# Listar jovens ativos
GET /items/Jovens?filter[status][_eq]=ativo

# Listar pulsos pendentes de um jovem
GET /items/pulsos_jovens?filter[jovem_id][_eq]=123&filter[status][_eq]=pendente

# Buscar matrículas de um jovem
GET /items/Matriculas?filter[Jovens_id][_eq]=123

# Atualizar matrícula
PATCH /items/Matriculas/456
{
  "certificate_status": "available",
  "certificate_url": "https://moodle.leapy.com/..."
}
```

### GraphQL

```graphql
query GetPulsosAtivos {
  pulsos_jovens(filter: { status: { _eq: "pendente" } }) {
    id
    pulso
    jovem_id {
      Name
      email_pessoal
    }
    data_vencimento
  }
}
```

## Flows

Flows são automações internas do Directus:

### Trigger Types

| Tipo | Descrição |
|------|-----------|
| **Event Hook** | Dispara em eventos CRUD |
| **Schedule** | Execução agendada |
| **Manual** | Executado manualmente |
| **Webhook** | Recebe webhooks externos |

### Exemplo: Notificar ao Criar Talento

```mermaid
flowchart LR
    A[Event: items.create] --> B{Collection = talents?}
    B -->|Sim| C[Preparar Dados]
    C --> D[Enviar Webhook Inngest]
```

### Operações Disponíveis

- **Condition**: Lógica condicional
- **Request**: Chamadas HTTP
- **Transform**: Transformar dados
- **Run Script**: Executar código
- **Send Notification**: Notificações internas

## Permissões

### Papéis Pré-definidos

| Papel | Acesso |
|-------|--------|
| `administrator` | Acesso total |
| `support` | Leitura + edições limitadas |
| `viewer` | Apenas leitura |

### Configuração de Permissões

1. Acesse **Settings > Roles & Permissions**
2. Selecione ou crie um papel
3. Configure permissões por collection:
   - Create / Read / Update / Delete
   - Campos visíveis
   - Filtros de dados

## Boas Práticas

1. **Nunca edite em produção** sem backup
2. **Use filtros** para limitar dados visíveis
3. **Documente alterações** em collections
4. **Teste flows** em staging primeiro
5. **Monitore performance** de queries
6. **Use endpoint customizado com critério**: priorize `/items` para CRUD e crie extensão de endpoint quando houver necessidade de contrato compartilhado e otimização comprovada

## Quando criar endpoint customizado no Directus

Use endpoint customizado no Directus quando:

- o contrato será consumido por múltiplos clientes/sistemas;
- a composição depende principalmente de dados e permissões do próprio Directus;
- existe necessidade real de reduzir múltiplas leituras para uma resposta otimizada;
- o time está preparado para manter extensões (versionamento, observabilidade e suporte).
