title: "Runbook: Pulso Não Ativado"
description: "Procedimento para resolver pulsos de jovens ou lideranças que não passaram para status pendente"
---

## Descrição do Problema

Um pulso agendado (`status = 'agendado'`) não foi ativado para `status = 'pendente'` na data de aplicação prevista. Os destinatários não receberam notificação e o pulso permanece invisível para resposta.

## Sintomas

- Pulso com `data_aplicacao <= HOJE` mas `status = 'agendado'` em `pulsos_jovens` ou `Pulsos`
- Jovem ou liderança não recebeu email/WhatsApp de liberação
- Dashboard do RH não mostra o pulso em aberto

## Diagnóstico

### 1. Verificar Status do Pulso de Jovem

```sql
-- Pulsos de jovens que deveriam estar pendentes
SELECT id, pulso, jovem_id, status, data_aplicacao, data_vencimento
FROM "public"."pulsos_jovens"
WHERE status = 'agendado'
  AND data_aplicacao <= CURRENT_DATE
ORDER BY data_aplicacao ASC
LIMIT 50;
```

**Se retornar resultados**: o job `updateStatusPulsosJovensParaPendentes` não executou ou falhou.

### 2. Verificar Status do Pulso de Liderança

```sql
-- Pulsos de lideranças que deveriam estar pendentes
SELECT id, pulso, jovem_id, lideranca_id, status, data_aplicacao
FROM "public"."Pulsos"
WHERE status = 'agendado'
  AND data_aplicacao <= CURRENT_DATE
ORDER BY data_aplicacao ASC
LIMIT 50;
```

### 3. Verificar Jobs no Inngest

Acesse o **Inngest Dashboard** > Functions e verifique:

| Job | Tipo | Horário |
|-----|------|---------|
| `updateStatusPulsosJovensParaPendentes` | Cron | Diário 8h |
| `updateStatusPulsosLiderancasToPendente` | Event | On-demand |

**Verificar:**
- O cron executou hoje?
- O run mais recente falhou? Em qual step?
- Há runs na fila não processados?

### 4. Verificar Configurações de Agendamento

```sql
-- Settings de agendamento habilitados
SELECT id, account_id, profile, enabled, next_pulso_date, cadence_interval
FROM "public"."pulsos_schedule_settings"
WHERE enabled = true
  AND next_pulso_date <= CURRENT_DATE
ORDER BY next_pulso_date ASC;
```

Se `next_pulso_date` está no passado e os pulsos não foram criados, o job `createPulsosJovens` também pode ter falhado antes.

## Causa Raiz Comum

| Causa | O que verificar |
|-------|----------------|
| Cron não disparou (Inngest) | Dashboard > Scheduled Functions |
| Step de update falhou | Logs do run específico no Inngest |
| `pulsos_schedule_settings` desabilitado | `enabled = false` no banco |
| Pulso criado com data futura incorreta | `data_aplicacao` no futuro |

## Resolução

### Opção 1: Forçar atualização via SQL (emergência)

```sql
-- ATENÇÃO: execute em staging primeiro. Revise os IDs antes de aplicar.
UPDATE "public"."pulsos_jovens"
SET status = 'pendente'
WHERE status = 'agendado'
  AND data_aplicacao <= CURRENT_DATE;
```

Após executar, acione manualmente as notificações:

```json
// Inngest Dashboard > Send Event
{
  "name": "backoffice/pulsos_jovens/lembrete-to-jovens.send",
  "data": {
    "pulso_ids": [12345, 12346]
  }
}
```

### Opção 2: Re-acionar job via Inngest

No Inngest Dashboard, acesse o job `updateStatusPulsosJovensParaPendentes` e clique em **Invoke** para executar fora do cron.

### Opção 3: Alterar via Backoffice Directus

1. Acesse Directus > `pulsos_jovens`
2. Filtre por `status = agendado` e `data_aplicacao <= hoje`
3. Altere `status` para `pendente` nas linhas afetadas

<Warning>
  Alterar o status diretamente no Directus **não dispara notificações automaticamente**. Após a correção, envie os lembretes manualmente via Inngest (Opção 1, segundo passo).
</Warning>

## Verificação Pós-Correção

```sql
-- Confirmar que não há mais pulsos agendados com data passada
SELECT COUNT(*)
FROM "public"."pulsos_jovens"
WHERE status = 'agendado'
  AND data_aplicacao <= CURRENT_DATE;
-- Resultado esperado: 0
```

- Jovens receberam email/WhatsApp de liberação
- Dashboard do RH mostra os pulsos em aberto

## Prevenção

1. Monitorar cron `updateStatusPulsosJovensParaPendentes` no Inngest — alerta se falhar
2. Query de sanidade diária: pulsos com `data_aplicacao <= hoje` e `status = agendado`
3. Alertas para crons do Inngest com falha em produção

## Escalação

Se o problema persistir ou múltiplas empresas forem afetadas:

1. Verificar saúde geral do Inngest e do banco (Supabase)
2. Abrir incidente se mais de 2 empresas afetadas simultaneamente
3. Acionar engenharia — verificar logs de erros detalhados no Inngest

## Contexto Técnico

O sistema de Pulsos usa status `agendado → pendente → respondido/vencido`. A transição `agendado → pendente` é feita exclusivamente pelo job cron `updateStatusPulsosJovensParaPendentes` (diário, 8h). Para lideranças, a transição é via evento `backoffice/pulsos.change_status`.

Veja documentação completa em [Jobs Inngest de Pulsos](/documentation/domains/pulses/jobs-inngest).
