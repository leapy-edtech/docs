## POST /matchmaker/search/stream

Busca candidatos com feedback em tempo real via **Server-Sent Events (SSE)**. Retorna eventos conforme cada passo do pipeline é executado.

<Tip>
  Este é o endpoint recomendado para uso no frontend. O streaming proporciona uma UX muito melhor, mostrando o progresso em tempo real.
</Tip>

## Request

```bash
POST /matchmaker/search/stream
Content-Type: application/json
```

### Body

| Campo | Tipo | Obrigatório | Descrição |
|-------|------|-------------|-----------|
| `query` | string | Sim | Query do usuário (texto, URL ou misto) |
| `limit` | integer | Não | Máximo de candidatos retornados (1-100, padrão: 10) |
| `department` | string | Não | Filtro por departamento |

## Response

**Content-Type**: `text/event-stream`

### Tipos de Eventos

| Tipo | Descrição | Quando |
|------|-----------|--------|
| `step` | Progresso de uma etapa | Durante o processamento |
| `result` | Resultado final completo | Ao final do processamento |
| `error` | Erro durante processamento | Quando algo falha |

### Formato dos Eventos

Cada evento segue o formato SSE padrão:

```
data: {"type": "step", "message": "mensagem"}

data: {"type": "result", "data": {...}}

data: [DONE]
```

### Eventos para Busca com URL

Quando a query contém uma URL, eventos adicionais de scraping são emitidos:

```
data: {"type":"step","message":"🌐 Capturando dados da página..."}

data: {"type":"step","message":"⏳ Ainda estamos trabalhando..."}

data: {"type":"step","message":"📄 Extraindo conteúdo..."}

data: {"type":"step","message":"✅ URL scrapeada com sucesso"}
```

Se o scraping falhar:

```
data: {"type":"step","message":"⚠️ Não foi possível analisar a URL enviada. Verifique se ela está correta ou copie e cole a descrição da vaga."}
```

## Consumindo no Frontend

### JavaScript/TypeScript (fetch + ReadableStream)

```typescript
async function searchWithStream(query: string) {
  const response = await fetch('/api/matchmaker/search/stream', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${accessToken}`
    },
    body: JSON.stringify({ query, limit: 10 })
  });

  const reader = response.body?.getReader();
  const decoder = new TextDecoder();
  let buffer = '';

  while (true) {
    const { done, value } = await reader.read();
    if (done) break;

    buffer += decoder.decode(value, { stream: true });
    const lines = buffer.split('\n');
    buffer = lines.pop() || '';

    for (const line of lines) {
      if (line.startsWith('data: ')) {
        const data = line.slice(6);
        if (data === '[DONE]') continue;

        const message = JSON.parse(data);

        switch (message.type) {
          case 'step':
            console.log('Progresso:', message.message);
            break;
          case 'result':
            console.log('Resultado:', message.data);
            break;
          case 'error':
            console.error('Erro:', message.message);
            break;
        }
      }
    }
  }
}
```

### EventSource (alternativa)

```javascript
const eventSource = new EventSource('/matchmaker/search/stream');

eventSource.onmessage = (event) => {
  const data = JSON.parse(event.data);

  if (data.type === 'step') {
    console.log(data.message);
  }

  if (data.type === 'result') {
    console.log('Candidatos:', data.data.candidate_result.candidates);
    eventSource.close();
  }
};
```

## Evento `result` - Schema

O evento `result` contém o mesmo schema da [busca síncrona](/api-reference/agent-system/matchmaker-search), com a adição do campo `candidate_result`:

### candidate_result

| Campo | Tipo | Descrição |
|-------|------|-----------|
| `candidates` | array | Lista de candidatos |
| `total_found` | integer | Total encontrados |
| `strategy_used` | string | Estratégia usada: `"exact"`, `"exact+llm_rerank"` |
| `skills_searched` | string[] | Skills usadas na busca |
| `execution_time` | float | Tempo do matching em segundos |

## Headers de Resposta

| Header | Descrição |
|--------|-----------|
| `Content-Type` | `text/event-stream` |
| `Cache-Control` | `no-cache` |
| `Connection` | `keep-alive` |
| `X-Accel-Buffering` | `no` (desativa buffering nginx) |
