## Visão geral

`POST /api/validate-user-email` verifica se um email já está em uso no sistema, chamando `Directus POST /validation/validate-email`. Usado em fluxos de onboarding para evitar duplicidade antes de submeter o formulário completo.

---

## POST /api/validate-user-email

### Autenticação

Sessão NextAuth com `accessToken`.

### Corpo

| Campo | Tipo | Obrigatório | Descrição |
|-------|------|-------------|-----------|
| `user_id` | `string` | sim | UUID do usuário sendo criado/atualizado (pode ser temporário) |
| `email` | `string` | sim | Email a verificar |

### Resposta `200` (email disponível)

```json
{
  "data": { "available": true }
}
```

### Resposta `200` (email em uso)

```json
{
  "data": { "available": false, "message": "Email já cadastrado" }
}
```

### Erros

| Status | Descrição |
|--------|-----------|
| `400` | JSON inválido, `user_id` ou `email` ausentes, ou email em uso (retornado pelo Directus) |
| `401` | Sessão inválida ou `accessToken` ausente |
| `4xx/5xx` | Erro retornado pelo Directus com passthrough de status |
