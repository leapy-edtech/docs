## Visão Geral

A API de Pulsos de RH permite coletar feedback do time de RH sobre a parceria com a Leapy, uso da plataforma e satisfação geral.

**Collection**: `pulsos_rh`  
**Base URL**: `/items/pulsos_rh`

## Schema

### Campos Principais

| Campo | Tipo | Descrição |
|-------|------|-----------|
| `id` | integer | Identificador único |
| `pulso` | integer | Número do pulso |
| `account` | uuid | ID da empresa/conta |
| `user` | uuid | ID do usuário RH |
| `email` | string | Email do respondente |
| `status` | string | Status atual |
| `data_aplicacao` | datetime | Data de disponibilização |
| `data_vencimento` | datetime | Data limite |
| `data_resposta` | datetime | Data de resposta |
| `respondido_por` | string | Nome do respondente |
| `ultimo_pulso` | boolean | Se é o último pulso |

### Campos de Avaliação

#### Satisfação (Escala 0-10)

| Campo | Descrição |
|-------|-----------|
| `nps_leapy` | NPS geral da Leapy |
| `report_e_dados_da_leapy` | Qualidade dos reports |
| `comunicacao_leapy` | Qualidade da comunicação |

#### Uso e Adoção

| Campo | Valores |
|-------|---------|
| `uso_da_plataforma_leapy` | "Muito Frequente", "Frequente", "Pouco", "Nunca" |
| `utiliza_outra_instituicao` | "Sim", "Não" |
| `nps_outra_insituicao` | NPS da concorrente (0-10) |
| `probabilidade_recontratacao` | "Muito Provável", "Provável", "Pouco Provável", "Improvável" |

#### Feedbacks

| Campo | Descrição |
|-------|-----------|
| `destaques_leapy` | Principais pontos positivos (json array) |
| `sucesso_do_programa` | Fatores de sucesso (json array) |
| `sugestoes` | Sugestões de melhoria (text) |
| `sente_falta` | O que sente falta (text) |
| `melhorias_plataforma` | Melhorias para plataforma (text) |

## Endpoints

### Listar Pulsos de RH

```bash
GET /items/pulsos_rh
```

<CodeGroup>

```bash cURL
curl -X GET "https://backoffice.leapy.com/items/pulsos_rh?filter[account][_eq]=uuid-da-empresa&filter[status][_eq]=pendente" \
  -H "Authorization: Bearer SEU_TOKEN"
```

```typescript TypeScript
const response = await fetch(
  'https://backoffice.leapy.com/items/pulsos_rh?filter[account][_eq]=uuid-da-empresa&filter[status][_eq]=pendente',
  {
    headers: {
      'Authorization': 'Bearer SEU_TOKEN'
    }
  }
);

const { data } = await response.json();
```

</CodeGroup>

**Resposta:**

```json
{
  "data": [
    {
      "id": 3456,
      "pulso": 2,
      "account": "uuid-da-empresa",
      "email": "rh@empresa.com",
      "status": "pendente",
      "data_aplicacao": "2024-02-01T09:00:00Z",
      "data_vencimento": "2024-02-15T23:59:59Z",
      "ultimo_pulso": false
    }
  ]
}
```

### Criar Pulso

```bash
POST /items/pulsos_rh
```

<CodeGroup>

```json JSON
{
  "pulso": 2,
  "account": "uuid-da-empresa",
  "user": "uuid-do-usuario",
  "email": "rh@empresa.com",
  "status": "agendado",
  "data_aplicacao": "2024-02-15T09:00:00Z",
  "data_vencimento": "2024-02-29T23:59:59Z",
  "ultimo_pulso": false
}
```

```typescript TypeScript
const response = await fetch(
  'https://backoffice.leapy.com/items/pulsos_rh',
  {
    method: 'POST',
    headers: {
      'Authorization': 'Bearer SEU_TOKEN',
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      pulso: 2,
      account: 'uuid-da-empresa',
      user: 'uuid-do-usuario',
      email: 'rh@empresa.com',
      status: 'agendado',
      data_aplicacao: '2024-02-15T09:00:00Z',
      data_vencimento: '2024-02-29T23:59:59Z',
      ultimo_pulso: false
    })
  }
);
```

</CodeGroup>

## Filtros Úteis

### Por Probabilidade de Recontratação

```bash
GET /items/pulsos_rh?filter[probabilidade_recontratacao][_eq]=Muito Provável&filter[status][_eq]=respondido
```

### Últimos Pulsos

```bash
GET /items/pulsos_rh?filter[ultimo_pulso][_eq]=true&sort=-data_aplicacao
```

### Por Range de NPS

```bash
GET /items/pulsos_rh?filter[nps_leapy][_gte]=8&filter[status][_eq]=respondido
```

### Comparar com Concorrentes

```bash
GET /items/pulsos_rh?filter[utiliza_outra_instituicao][_eq]=Sim&fields=nps_leapy,nps_outra_insituicao
```

## Agregações

### NPS Médio por Empresa

```bash
GET /items/pulsos_rh?aggregate[avg]=nps_leapy&groupBy[]=account&filter[status][_eq]=respondido
```

**Resposta:**

```json
{
  "data": [
    {
      "account": "uuid-empresa-1",
      "avg": { "nps_leapy": 8.5 }
    },
    {
      "account": "uuid-empresa-2",
      "avg": { "nps_leapy": 9.2 }
    }
  ]
}
```

### Taxa de Resposta

```bash
GET /items/pulsos_rh?aggregate[count]=id&groupBy[]=status
```

### Distribuição de Uso

```bash
GET /items/pulsos_rh?aggregate[count]=id&groupBy[]=uso_da_plataforma_leapy&filter[status][_eq]=respondido
```

## Análise de Destaques

O campo `destaques_leapy` é um array JSON:

```json
{
  "destaques_leapy": [
    "Suporte rápido e eficiente",
    "Talentos bem treinados",
    "Plataforma fácil de usar",
    "Relatórios completos"
  ]
}
```

Buscar pulsos com destaque específico:

```bash
GET /items/pulsos_rh?filter[destaques_leapy][_contains]=Suporte rápido
```

## Boas Práticas

### Periodicidade

<Tip>
  Recomenda-se aplicar pulsos de RH trimestralmente para não sobrecarregar o time e capturar mudanças ao longo do tempo.
</Tip>

### Campos Obrigatórios

```json
{
  "account": "obrigatório",
  "user": "obrigatório",
  "email": "obrigatório",
  "pulso": "obrigatório",
  "status": "obrigatório",
  "data_aplicacao": "obrigatório",
  "data_vencimento": "obrigatório"
}
```

### Valores Permitidos

```json
{
  "uso_da_plataforma_leapy": ["Muito Frequente", "Frequente", "Pouco", "Nunca"],
  "probabilidade_recontratacao": ["Muito Provável", "Provável", "Pouco Provável", "Improvável"],
  "utiliza_outra_instituicao": ["Sim", "Não"]
}
```

### Interpretação do NPS

| NPS | Classificação |
|-----|---------------|
| 9-10 | Promotor |
| 7-8 | Neutro |
| 0-6 | Detrator |

eNPS = % Promotores - % Detratores
